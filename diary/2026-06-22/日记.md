# 2026-06-22 工作日记

## jhin.ink 域名配置（主要战斗）

### 背景
- 域名在**爱名网（22.cn）**注册，要把作品集挂到自定义域名上
- 作品集托管在 **Cloudflare Pages**：jin-chengxu-portfolio.pages.dev

### 全流程

1. **爱名网改 DNS**：凌晨把 nameserver 从爱名网默认改为 Cloudflare 的 `carioca.ns.cloudflare.com` + `tanner.ns.cloudflare.com`
2. **Cloudflare 添加站点**：jhin.ink 出现在 Cloudflare 账号中，状态 pending（等待注册局验证 NS 变更）
3. **DNS 记录配置**：通过 Cloudflare API 添加了两条 CNAME
   - `jhin.ink` → `jin-chengxu-portfolio.pages.dev`
   - `www.jhin.ink` → `jin-chengxu-portfolio.pages.dev`
4. **Pages 自定义域名**：尝试在 Cloudflare Pages 项目添加 jhin.ink，遇到 DNS 转移提示，完成 Connect a domain 流程
5. **反复验证**：今天 DNS 还未全球生效，Zone 状态 still pending，注册局 NS 变更等待中

### 当前状态
- Cloudflare DNS 记录：✅ 两条 CNAME 已配置
- Cloudflare Zone：⏳ pending（等待 .ink 注册局确认 NS 变更，24-48h）
- Pages 自定义域名：⏳ 等 Zone active 后添加
- 访问测试：❌ curl 仍返回 HTTP 000

### 学到的东西
- 域名 NS 变更的全球传播是瓶颈，急不来
- Cloudflare API Token 权限要精确（DNS token 不能操作 Pages）
- 爱名网的 NS 修改需要走注册局，不是即时的
